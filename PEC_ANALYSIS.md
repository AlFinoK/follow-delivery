# Калькулятор доставки (модель ПЭК) — анализ и реализация

Источник: `https://pecom.ru/services-are/shipping-request/`
Метод анализа: Playwright — реверс боевого расчётчика ПЭК. Город отправления зафиксирован = **Алматы** (филиал ПЭК `8112fb85-5a96-11e4-94e4-00155d9d920f`), как требует ТЗ.

> **Важно:** у ПЭК НЕТ клиентской формулы расчёта — вся тарификация считается на их сервере
> (JSON-RPC `Order.Request.Update`, `action: "calculate"`, ответ `result.calculate.auto.basePrice`).
> Поэтому «скопировать формулу» = снять реальные тарифы по каждому направлению и воспроизвести
> их офлайн. Что и сделано: тарифы 169 городов сняты напрямую с расчётчика ПЭК и зашиты в
> `lib/calculator/directions.json`. Калькулятор **автономный** — в рантайме к ПЭК не обращается.

---

## 1. Как сняты данные (одноразовый сбор через Playwright)

Реальный API ПЭК (всё — `POST /ajax/`, JSON-RPC 2.0, требуется Bitrix `sessid` живой сессии):

| Метод | Назначение |
|---|---|
| `Basic.Department.Filter` (`bounds`, `departmentTypes:"all"`, `needShortedData:1`) | весь справочник из 629 терминалов: `{id, code, name, address, coordinates, departmentType}` |
| `Order.Department.NameFilter` (`address`, `type:"to"`, `bid`) | резолв города → терминал (для адресного поиска) |
| `Order.Request.Update` (`action:"calculate"`) | расчёт: `result.calculate.auto.basePrice` (склад-склад «Автоперевозка»), `countDays` (срок, раб. дней) |

Из 629 терминалов взяты **главные грузовые отделения РФ** (`departmentType === "4"`, адрес РФ) — это
по одному терминалу на город, **169 городов**. Тип `"1"` — это ПВЗ/СДЭК-пункты (для сборного груза
склад-склад не нужны), `"5"` — 3PL-склады, `"agentPoint"` — зарубежные агенты.

Для каждого города прогнаны расчёты по сетке весов и объёмов (Алматы → город), записаны
`basePrice` и `countDays`. Курс анти-бота (QRATOR + Yandex SmartCaptcha) не позволяет дёргать ПЭК
с сервера в рантайме — поэтому данные сняты заранее из браузера и зашиты в репозиторий.

## 2. Основная формула (воспроизведена точно)

```
стоимость (₽, склад-склад) = max( тариф_по_весу(вес_кг) , тариф_по_объёму(объём_м³) )
```

ПЭК берёт **бо́льшую** из двух стоимостей — по фактическому весу и по объёму. У каждого направления
**две независимые кривые** (по весу и по объёму) — отдельной «плотности» в модели нет (старое
приближение «×280 кг/м³» давало 2–4% ошибки, поэтому заменено на реальную кривую объёма).

### Кривая по весу — структура (одинаковая по всем городам, отличаются только цены)

- **≤ 35 кг — ступенчатый минимальный тариф** (внутри бакета цена постоянна). Границы бакетов:
  `≤1, (1,3], (3,5], (5,15], (15,35]`. Пример Москвы: 1150 / 1500 / 1830 / 1970 / 2360 ₽.
- **> 35 кг — кусочно-линейная убывающая ставка ₽/кг** с изломами. Пример Москвы: ~56.6 ₽/кг около
  50–100 кг, плавно вниз до ~42.4 ₽/кг при ≥ 5 т.

Узлы, на которых сняты тарифы (`lib/calculator/config.ts`):
- вес-ступени: `[1, 3, 5, 15, 35]` (интерполяция — ceil к границе бакета);
- вес-линейная часть: `[35, 50, 75, 100, 125, 150, 200, 250, 300, 500, 600, 1000, 1500, 2000, 5000, 20000]`;
- объём: `[0.1, 0.25, 0.5, 0.75, 1, 2, 3, 5, 10, 20]` (линейная интерполяция; < 0.1 м³ — линейно через ноль).
- свыше последнего узла — продление по ставке последнего участка.

### Габариты и места
Расчёт «по габаритам мест» = сумма: `общий_объём = Σ(Д×Ш×В×кол-во)`, `общий_вес = Σ(вес×кол-во)`,
затем та же формула. Единицы — метры/сантиметры (клиентский пересчёт).

### Сроки доставки
`countDays` — диапазон рабочих дней `[мин, макс]` по направлению (Москва 9–14, Владивосток дольше и т.д.).

## 3. Точность (валидация на боевом ПЭК)

1014 сравнений (169 городов × 6 промежуточных, не-узловых точек: 7/120/800 кг, 0.3/1.5 м³, смешанный 300 кг+2 м³):

- средняя ошибка **1.0%**, медиана **0.7%**, p90 **2.5%**, p99 **5.1%**, максимум **6%**;
- в пределах 2% — **84%** кейсов, в пределах 5% — **99%**;
- ступенчатая зона (лёгкий груз) и узлы сетки — **0%** (точно).

Остаточные ~5% — только на «чистом малом объёме» (0.3 м³ при near-нулевом весе) в мелких городах;
для реального груза (вес+объём вместе) ошибка ~1–2%.

---

## 4. Что НЕ входит в базовую стоимость (по решению заказчика)

Считается только **база склад-склад («Автоперевозка»)**, в **тенге** (конвертация ₽→₸ по курсу
Нацбанка РК, см. `/api/rate`; запасной курс в конфиге). Не входят (есть у ПЭК отдельными услугами):
- страхование груза и оргсбор; забор/доставка «по адресу»; экспресс-перевозка; упаковка.

Исключённые направления (ДНР/ЛНР/Крым-санкционные и т.п.) — **наш блок-лист** (`EXCLUDED_PATTERNS`),
такие города скрыты из справочника.

---

## 5. Где лежит и как править (без изменения кода)

- `lib/calculator/directions.json` — **169 городов**: `{ name, region, code, lat, lng, days:[min,max], w:{вес→₽}, v:{объём→₽} }`.
  Цены правятся прямо здесь (сид-данные с ПЭК на дату сбора; заказчик корректирует свободно).
- `lib/calculator/config.ts` — узлы сеток, блок-лист, валюта, запасной курс.
- `lib/calculator/engine.ts` — чистые функции расчёта (`priceByWeight`, `priceByVolume`, `calcShipment`).
- UI: `components/calculator/*`, страницы `app/calculator` (публичная) и `app/admin/calculator`.

### Как пересобрать тарифы заново
Открыть калькулятор ПЭК в браузере (живая сессия → `sessid`), снять `Basic.Department.Filter` +
`Order.Request.Update`/`calculate` по сетке весов/объёмов на каждый город, собрать `directions.json`
в том же формате. (Серверный прокси невозможен из-за QRATOR + SmartCaptcha.)

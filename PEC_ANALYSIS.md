# Анализ калькулятора ПЭК (pecom.ru) — логика расчёта стоимости

Источник: `https://pecom.ru/services-are/shipping-request/`
Метод анализа: Playwright — реверс реального API расчёта (`POST /ajax/`, JSON-RPC `Order.Request.Update`, `action: "calculate"`; справочник филиалов — `Order.Department.NameFilter`). Прогнаны десятки контролируемых кейсов: развёртки по весу, по объёму, смешанные грузы, габариты, направления, удалённые регионы, исключённые территории, страхование.

Город отправления зафиксирован = **Алматы** (филиал ПЭК `8112fb85-5a96-11e4-94e4-00155d9d920f`), как требует ТЗ.

---

## 1. Основная формула (проверено с точностью ~1–2%)

```
платный_вес = max( вес_кг , объём_м³ × ПЛОТНОСТЬ )
стоимость   = тариф_направления( платный_вес )
```

- **ПЛОТНОСТЬ = 280 кг/м³** — глобальная константа (одинакова для всех направлений; подтверждено: цена «по объёму» совпадает с ценой «по весу» при `V×280` в пределах 1–2%).
- Это и есть «соотношение объёма и веса» из ТЗ: ПЭК берёт **бо́льшую** из двух стоимостей — по фактическому весу и по объёмному весу. Объёмный вес = `объём × 280`.
- Эквивалентная формулировка ПЭК: `стоимость = max(по весу, по объёму)`.

### Тариф направления — убывающая ставка ₽/кг
Для каждого направления это кусочная кривая «вес → цена»:
- **минимальный тариф** (плато) для лёгких грузов — у Москвы ≈ 1150 ₽ за ≤1–5 кг;
- далее ступенчатый рост, около **56 ₽/кг** в районе 50–100 кг;
- ставка плавно убывает с весом (опт дешевле): Москва выходит на **~42 ₽/кг** при ≥10 т.
- Каждое направление — **самостоятельная кривая** (не множитель от Москвы), но форма у всех похожая. Дальние направления дороже: Уфа ≈31.6 ₽/кг, Москва ≈42.4, Владивосток ≈88, Якутск ≈133.

### Минимальный тариф
= значение кривой при ~1 кг (Москва 1150 ₽; более близкие/дешёвые города ~900 ₽).

---

## 2. Габариты и единицы измерения
- Расчёт «по габаритам мест» = точная сумма: `общий_объём = Σ(Д×Ш×В×кол-во)`, `общий_вес = Σ(вес×кол-во)`, затем та же основная формула. Проверено: несколько типов мест дают цену, идентичную вводу суммарных объёма/веса.
- Единицы: метры или сантиметры — простой клиентский пересчёт. Вес — кг.
- **Негабарит / длина места** базовую магистральную стоимость в этом калькуляторе НЕ меняет (отдельная услуга).

---

## 3. Сроки доставки
Календарные дни от даты сдачи (для понедельника), по направлениям. Примеры из Алматы:
Москва 12, Самара/Уфа 11, Екатеринбург 12, Казань 14, Новосибирск 15, Краснодар 15, Ростов 17, Сочи 17, Иркутск 18, Хабаровск 24, Владивосток 26, Калининград 28, Магадан 45, Южно-Сахалинск 50, Камчатка 56. Сроки сохраняем как у ПЭК.

---

## 4. Удалённые регионы
Обслуживаются, но дороже и дольше (Норильск, Магадан, Камчатка, Якутск — высокая ставка ₽/кг + большие сроки). Отдельной «формулы для удалёнки» нет — это просто их собственные дорогие кривые + длинные сроки.

---

## 5. Исключённые направления — это НАШЕ бизнес-правило, не ПЭК
ПЭК фактически возит в Донецк (≈7125 ₽), Крым (Симферополь/Севастополь), Грозный, даже Магадан. Луганск/Макеевка — терминал не находится. Поэтому исключение ДНР/ЛНР/Донецк/Луганск и т.п. реализуем сами через **редактируемый блок-лист**.

---

## 6. Доп. составляющие (вне основной стоимости, опционально)
- **Страхование груза**: `max(51 ₽, 0.1% × объявленная стоимость)`; «организация страхования» ≈ 0.42× от этого (ступенчато). В ядро расчёта ТЗ не входит.
- **Забор/доставка по адресу** (двери): отдельные тарифы (Алматы→Москва: 518.18 / 790 ₽).
- **Экспресс-перевозка**: ≈ +12% к «Стандарту» на лёгком грузе, больше на тяжёлом (отдельная кривая).

---

## 7. Что переносим в редактируемый конфиг (требование ТЗ)
- `density` (280 кг/м³);
- по каждому направлению: кривая «вес → цена» + срок доставки (сид: 54 города с реальными тарифами ПЭК);
- блок-лист исключённых направлений;
- (опц.) коэффициент экспресса, тарифы дверей, параметры страхования.

> Все числа — **сид-данные**, снятые с ПЭК на дату анализа. Это первый этап «повторить как есть»; далее заказчик корректирует тарифы/коэффициенты без правок кода.

## Где лежит конфиг (правится без кода)
- `lib/calculator/directions.json` — 54 города: `{ code, name, days, curve }`, где `curve` — карта «вес (кг) → цена (₽)» по сетке [1,3,5,10,16,25,35,50,75,100,200,500,1000,2000,5000,20000].
- `lib/calculator/config.ts` — `DENSITY_KG_PER_M3` (280), блок-лист `EXCLUDED_PATTERNS`, `FALLBACK_DIRECTION` (оценка для городов вне справочника), валюта.
- `lib/calculator/engine.ts` — чистые функции расчёта (платный вес, интерполяция кривой, суммирование мест).
